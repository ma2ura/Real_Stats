3.1 データを知る

経営学で用いられるデータは、公表されているデータや、研究者が自分で収集したデータなど、さまざまな種類があります。 経営関連のデータの多くは観察データ(observational data)であり、研究者がコントロールできない状況で収集されたデータです。 したがって、無重力の真空状態で行われる物理学の実験で得られたデータとは異なり、経営学のデータは、ゴミやノイズが多く混入しています。 そのため、分析の前にデータをクリーニングする必要もあります。 そういったゴミやノイズが含まれていることを前提として、自分の手元にあるデータについて理解することが重要です。

データの大きな特徴をつかむために、記述統計(descriptive statistics)を用います。 ここでは、

  • 観測値の数 n()
  • 平均値 mean()
  • 標準偏差 sd()
  • 最小値 min()
  • 最大値 max()

を確認することが重要です。 第1章のドーナツのデータを例にして、これらの記述統計を確認してみましょう。

load("data/DonutData.Rdata") # ドーナツのデータを読み込む
df <- data.frame(name, donuts, weight, child , male) # データフレームにまとめる
df |>
    select(weight, donuts) |> # donutsとweightの列を選択
    describe() |> # psychパッケージのdescribe関数で記述統計を計算
    select(n, mean, sd, min, max) |> # 必要な列だけを選択
    round(2) # 小数点以下2桁に丸める

このデータに含まれている男性と女性の人数も重要な要素なので、確認してみましょう。

table(df$male) |> knitr::kable()
Var1 Freq
0 4
1 9

データのグラフ化も異常な観測値の存在に気づくために重要な手段です。 散布図(scatter plot)を使って、変数間の関係を確認したり、箱ひげ図(boxplot)やヒストグラム(histogram)を使って、変数の分布を確認することもできます。

たとえば、ドーナツを食べた個数の分布をヒストグラムで確認してみましょう。

hist(df$donuts)
view(df)

hist()関数はデフォルトで、右閉左開区間でビンを作成しますが、例外的に最小値(ここでは0)だけは含まれることになっているので、 区間は、[0,5], (5,10], (10,15], (15,20], (20,25] のようになります。

この図から、ほとんどの人が0〜5個のドーナツを食べているのに対して、10個以上が1人、15個以上が1人、20個以上が1人いることが分かり、この3名がたくさんのドーナツを食べていることで、平均値が引き上げられていることが分かります。

箱ひげ図は次のように書きます。

boxplot(df$donuts)

ヒストグラムは基本関数hist()関数で、箱ひげ図は基本関数boxplot()関数で書くことができます。 より凝ったグラフを描きたい場合は、ggplot2パッケージを使い、 geom_histogram()関数でヒストグラムを、geom_boxplot()関数で箱ひげ図を描くことができます。

適切なグラフは一目でデータの特徴をつかむことができるので、分析の前にデータをグラフ化してみることをお勧めします。 ただし、3Dグラフは避けましょう。

3.2 再現性

科学的知識の確信は再現性(replication)です。 だれでも研究の詳細が分かれば、同じ結果を得られるようにすることが重要です。 その基準を満たすために、研究再現用ファイル(replication files)を作成して、

  • データの収集方法
  • 欠損データの処理方法
  • データの整理方法
  • 変数の定義や加工プロセスを書いたコードブック
  • 分析を行うためのコード

を保持することが重要です。 レポートや論文を書く際に、

  • 財務データや日経NEEDSから取得し、
  • 情報が欠損している観測値を削除し、
  • 連続変数については上下1％を切り落として外れ値を処理し、
  • 売上高や総資産は対数変換して、
  • 統計モデルを構築して、最小二乗法で分析を行った

といった分析のプロセスを書いておく必要があるのです。 分析に用いた変数の定義については、誰でも同じ変数が構築できるように、正確かつ厳密に書いておく必要があります。 たとえば、Warren (2025, TAR)という自発的開示の質とアナリストカバレッジの関係を分析した論文では、変数の定義一覧を付録(appendix)に掲載しています。以下の表はその一部ですが、定義が詳細に書かれていることが分かります。

変数名 定義
N_forecasts 企業が発表した経営者予測の数
Accuracy 経営者予測の精度。I/B/E/S Guidanceからの経営者予測とI/B/E/Sからの実際のEPSの絶対値の差を、期間開始時の株価で割り、100倍して算出。範囲予測が提供された場合は、範囲の上限を使用 (Ciconte et al. 2014)。
Accuracy Bias 経営者予測のバイアス。I/B/E/S Guidanceからの経営者予測とI/B/E/Sからの実際のEPSの差を、期首の株価で割り、100を乗じて算出。範囲予測が提供された場合は、範囲の上限を使用 (Ciconte et al. 2014)。
Horizon 予測日から期末の日数を365で割って算出。イベントの前後に複数の予測が発表された場合は、前期間と後期間の平均予測期間をとる。

のように変数名とその定義を表形式でまとめている。

研究再現用ファイルの作成は、研究結果の頑健性(robustness)を確認するためにも重要です。 変数を他の代理変数に置き換えたり、分析方法を変えたりしても、同じ結論が得られるかどうかを確認することができます。

3.3 統計ソフトウェア

ここでは、RとStataという2つの統計ソフトウェアを紹介します。 たとえば、CSVファイルを読み込み、データの中から一部の変数を取り出し、変数を対数変換してから回帰分析を行う、といった一連の流れを、StataとRでどのように書くのかを見てみましょう。

  • Stata：強力な統計分析機能をもつソフトウェアで、社会科学の分野で研究者に広く愛用されているが、値段が高いのが難点です。 しかし立命館大学の学生は在学中なら無料で利用できるので、一度使ってみることをオススメします。
Stataでの分析例
* データの読み込み
import "data.csv", clear
* データの前処理
keep id year sales assets debt industry
gen ln_assets = ln(assets)
* 回帰分析
reg sales ln_assets debt
* 作表
estimates store m1
esttab m1 using "results.rtf", replace b(3) t(3) r2 ar2 star(* 0.1 ** 0.05 *** 0.01)
  • R：無料で使えるオープンソースのプログラミング言語で、データ分析の世界ではPythonと並んで人気の言語ですが、ソースコードを書く必要があるため、ユーザーフレンドリーではありません。
Rでの分析例
pacman::p_load(tidyverse, modelsummary)
# データの読み込み
df <- read_csv("data.csv")
# データの前処理
analysis_data <- df |>
  select(sales, ln_assets, debt, industry) |>
  mutate(ln_assets = log(assets))
# 回帰分析
model_ols <- lm(sales ~ ln_assets + debt, data = analysis_data)
# 作表
modelsummary(
  model_ols,
  fmt = 3,                 # 小数点第3位まで表示
  estimate = "{estimate}{stars}", # 係数に有意水準の星を付与
  statistic = "std.error", # 括弧内は標準誤差（t値への変更も可能）
  gof_omit = "IC|Log|F",   # 不要な適合度指標を非表示にする
  output = "markdown"      # ファイル保存時は "table.docx" や "table.pdf"
)

ヒストグラムは基本関数`hist()`関数で、箱ひげ図は基本関数`boxplot()`関数で書くことができます。

```{r}
#| echo: false
res_hist <- hist(df$donuts, plot = FALSE)
breaks_val <- res_hist$breaks

# 2. ggplot2 で再現する
# boundary は hist() の挙動に合わせて調整し、breaks を直接指定します
g <- ggplot(df, aes(x = donuts)) +
  geom_histogram(
    breaks = breaks_val,
    color = "black", fill = "lightblue",
    closed = "right"      # hist(right = TRUE) と同じ挙動にする
  ) +
  labs(
    x = "Donuts",
    y = "Frequency"
  ) + theme_bw()

# 表示
print(g)
```
:::
