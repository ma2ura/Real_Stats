2  因果の探究

  • Show All Code
  • Hide All Code
  • View Source

他人や自分の説得させる証拠を提供したい。 ある現象の原因と結果を直接観察できないとき、データを用いて因果関係を探究する方法を学びます。 とはいえ、

\text{相関係数} \not = \text{因果関係}

であることを理解することが重要です。

本書では、原因を特定し、本当は何が重要で何が重要でないかを議論するために役立つ統計学を学びます。 そのために、第1章では、3つの重要な概念を紹介します。

  1. 特定化(specification) : 統計モデルを記述すること
  2. 統計的推論(statistical inference) : データからモデルのパラメータを確率的に評価すること
  3. 識別(identification) : 因果関係を特定できる統計的検定を設計すること

統計的推論と識別において2つの解決すべき課題も議論する。

  1. ランダム性(randomness) : データはランダムに生成されるため、偶然の要素が存在すること
  2. 内生性(endogeneity) : 説明変数と誤差項が相関していること

2.1 特定化 - 基本となるモデル

何が原因で何が結果なのかをみるために、

  1. 変数の特定
  2. 変数間の関係を数式で表現

という2つのステップを踏む必要があります。 とりわけ、結果を表す変数を従属変数(dependent variable)と呼び、原因を表す変数を独立変数(independent variable)と呼びます。

たとえば、結果を表す変数を売上高、原因を表す変数を広告宣伝費とし、この2つの変数間の関係を

\text{売上高} = a + b \times \text{広告宣伝費}

という数式で表現することを特定化といいます。 上の式を、a=10、b=0.5とすると、以下のようなグラフになります。

a <- 10
b <- 0.5
x <- seq(0, 100, by = 10)
y <- a + b * x
plot(x, y, type = "b", xlab = "広告宣伝費", ylab = "売上高")

テキストでは、ドーナツを食べた個数を独立変数、体重を従属変数とするモデルを特定化しています。 データハンドリングや作図のために、tidyverseパッケージを使用します。 pacmanパッケージのp_load関数を使って、必要なパッケージをインストール/ロードします。 load()関数を使って、ドーナツのデータDonutData.Rdataを読み込みます。 読み込んだデータをdata.frame()関数を使ってデータフレームdfにまとめます。 次に、ggplot2パッケージを使って、ドーナツを食べた個数と体重の関係を散布図で可視化します。

pacman::p_load(tidyverse)
load("data/DonutData.Rdata")
df <- data.frame(name, donuts, weight, child , male)
ggplot(df) + 
    aes(x = donuts, y = weight) + geom_point() + 
    # ラベルにnameを重ならないように表示
    geom_text(aes(label = name), vjust = -1, hjust = 0.5) +
    theme_bw() + labs(x = "ドーナツを食べた個数", y = "体重(kg)")

このとき、次のような関係を想定する。

\text{体重}_i = \beta_0 + \beta_1 \times \text{ドーナツを食べた個数}_i + \varepsilon_i

ここで、iは個人を表す番号、\beta_0は切片、\beta_1はドーナツを食べた個数の係数、\varepsilon_iは誤差項を表します。 関心があるのは、「ドーナツを食べた個数」が1単位増えたとき、「体重」がどれだけ増えるのかを意味する、\beta_1の値です。 上のモデルは、「ドーナツを食べた個数」と「体重」との間に、切片が\beta_0、傾きが\beta_1の線形関係があると仮定しています。 つまり、次のような直線を想定していることになります。

コードを表示
ggplot(df) + 
    aes(x = donuts, y = weight) + geom_point() + 
    # ラベルにnameを重ならないように表示
    geom_text(aes(label = name), vjust = -1, hjust = 0.5) +
    geom_smooth(method = "lm", se = FALSE, color = "blue") +
    # 誤差項を表す点線を追加
    geom_segment(aes(x = donuts, xend = donuts, y = weight, yend = predict(lm(weight ~ donuts, data = df))), 
                 linetype = "dashed", color = "red") +
    theme_bw() + labs(x = "ドーナツを食べた個数", y = "体重(kg)")

この図の黒い点が実際のデータを、青い線が特定化したモデルを表しており、実際のデータとモデルの乖離である誤差項を赤い点線で表しています。 この統計モデルが現実のデータを完全に説明できているなら、実際のデータはすべて青い直線上に位置するはずですが、実際にはモデルからズレた場所に点があるため、誤差が存在します。 つまり、「ドーナツを食べた個数」が「体重」に与える影響を完全に説明できていないということになり、説明できていない部分が誤差項\varepsilon_iに含まれていることになります。

2.2 2つの大きな課題 : ランダム性と内生性

2.2.1 ランダム性と推論

偶然にたまたま生じてしまっている結果と、偶然ではなく必然的に生じている結果を区別することは重要です。 いま得られた結果が、本当は何の関係もないのに、偶然にたまたま関係があるように見えているのか、それとも因果関係によって生じているのかを区別するため、たまたま得られた結果ではない、という確信を得るための方法があります。 この方法は、統計的推論(statistical inference)、あるいは単に推論と呼ばれています。

統計学におけるもう一つの重要な課題は、内生性(endogeneity)と呼ばれる問題です。
